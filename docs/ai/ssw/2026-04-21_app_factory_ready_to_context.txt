# SSW – App Factory / Neo – 2026-04-21 (레디투맥락 세션)

## 세션 목적
- App Factory / Neo / 헌법 v4 작업을 다음 세션에서도 끊김 없이 이어가기 위한
  “레디투맥락(Ready-to-Context)” 기준선 정의.
- 향후 세션에서 헐크/커서가 맥락을 잃지 않도록, 규칙 레이어와 세션 운용 방식의 뼈대를 합의.


## 오늘 합의한 핵심 포인트

- 헌법 레포 역할
  - `github.com/ollanvin/app-factory-constitution`는 App Factory 전체의
    공통 헌법/스키마 중앙 원격이다.
  - 네오(executing-OS)와 외부 워커/컨트롤 플레인은 모두 이 레포의
    `shared/constitution`을 기준으로 policy-as-code를 공유한다.
  - realrun-report는 이 레포의 커밋/버전을 기준으로 "어느 헌법 버전에서 돌았는지"
    추적할 수 있어야 한다.

- 네오 레포 역할
  - `github.com/ollanvin/executing-OS`는 로컬 OS / 실행기 레포다.
  - 여기서 헌법은 위 constitution 레포를 체크아웃한 경로를 통해 읽는다
    (기본은 `NEO_CONSTITUTION_ROOT`로 주입).

- 맥락 관리 3 레이어 구조
  1) Rules (프로젝트 고정 규칙)
     - 예: `docs/ai/ai_rules_neo.md`
     - 이 프로젝트에서 AI/에이전트가 절대 바꾸면 안 되는 상수:
       - 헌법 v4 기준 (constitution 레포 참조)
       - 에뮬레이터 우선, 실기기 런은 예외
       - ADB/화면 캡처 툴은 이미 구현되어 있으며, 특정 경로에 존재
       - "새 아카이브 파일을 막 만들어내지 말고, 아래 정해진 파일만 갱신한다"는 원칙
     - 최종 편집권은 사람(대표님). AI는 수정 제안만 한다.

  2) Checkpoint (현재 진도 한 장 요약)
     - 예: `docs/ai/neo_session_checkpoint.md`
     - 내용:
       - 지금까지 구현된 것 3~10줄
       - 다음 세션에서 할 일 3~5줄
       - 이번 세션에서 AI에게 반드시 알려야 할 전제 몇 줄
     - 세션 끝날 때 헐크가 초안 제안 → 대표님이 확인/수정 후 저장.
     - 다음 세션 시작 시, 이 파일 내용을 그대로 첫 프롬프트에 붙여 넣는다.

  3) SSW / Archive (세부 히스토리)
     - 예: `docs/ai/ssw/날짜_주제.md`
     - 각 세션에서 오간 결정/실험/배경을 길게 써두는 히스토리.
     - 길어도 괜찮고, 사람이 읽기 위한 문서. 헐크는 필요할 때만 특정 부분을 참고.

- 세션 운용 원칙 (짧게, 한 테마씩)
  - 한 세션은 대략 30~60분, 한 작업 단위만 처리:
    - 예: "규칙파일 잠그기", "ADB/에뮬 배선", "리얼런 리포트 포맷 튜닝" 등.
  - 한 세션에서:
    - 새 파일 1~3개 생성, 수정 파일 3~7개 정도를 상한선으로 본다.
  - 세션이 길어지면 헐크가 먼저:
    - "요약 만들고 세션 끊자"를 제안하고,
    - Checkpoint와 SSW를 갱신한 뒤 새 세션으로 넘어간다.


## 헐크/커서 사용 규칙 (다음 세션부터 적용)

- AI는 임의로 새로운 SSW/요약 파일을 만들지 않는다.
  - 규칙은 `docs/ai/ai_rules_neo.md`에서만 변경 제안.
  - 현재 진도는 `docs/ai/neo_session_checkpoint.md`만 갱신.
  - 세션 히스토리는 `docs/ai/ssw/YYYY-MM-DD_*.md` 아래에만 추가.

- 새 세션 시작 시, 사람(대표님)은 항상 다음 정보를 첫 프롬프트에 포함한다.
  - 프로젝트/레포:
    - executing-OS, app-factory-constitution
  - Rules 파일 경로:
    - `docs/ai/ai_rules_neo.md`
  - Checkpoint 파일 경로:
    - `docs/ai/neo_session_checkpoint.md`
  - 이번 세션 목표 한 줄:
    - 예: "이번 세션: ai_rules_neo.md 규칙파일 내용 잠그기."

- AI는 세션 초반에 반드시:
  - Rules 파일을 읽고 이 세션에서 지켜야 할 상수들을 정리한 뒤,
  - Checkpoint 파일을 읽고 "지금까지 진도 + 이번 세션 목표"를 재확인해야 한다.


## 오늘 세션 진도 요약

- 맥락 문제를 줄이기 위한 3 레이어 구조(Rules / Checkpoint / SSW)를 합의했다.
- constitution 레포와 executing-OS 레포의 역할을 명확히 분리했다.
- 다음 세션의 목표를 "규칙파일(ai_rules_neo.md) 내용 잠그기"로 확정했다.
- 세션 전환 프로토콜:
  - 세션 끝에는 Checkpoint 10줄 + 다음 세션 목표 1줄을 남기고,
  - 새 세션은 그 Checkpoint와 Rules/SSW 경로를 함께 주는 방식으로 시작한다.

  - "아카이브 파일을 새로 만들지 말고, 정해진 파일만 갱신할 것"이라는 맥락 관리 규칙.
- 세션 범위는 규칙파일 작성/수정까지로 제한하고,
  - 실제 코드 변경(ADB 배선 등)은 그 다음 세션에서 처리한다.
